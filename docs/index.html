Abstract

Robotic assembly systems traditionally require substantial manual engineering effort to integrate new tasks, adapt to new environments, and improve performance over time. This paper presents a framework for autonomous integration and continuous improvement based on Skill Graph representations.

A Skill Graph organizes robot capabilities as verb-based skills and explicitly links semantic descriptions with executable policies, preconditions, postconditions, and evaluators. This lets the system plan at the semantic level while still grounding execution in robot controllers and perception modules.

Framework Overview

The central idea is to make every robot capability explicit. Instead of hiding task logic inside one-off scripts, AIDF represents skills, planning constraints, execution interfaces, and evaluation criteria in one graph structure.

Skill Graph representation

Each skill is represented by a verb, applicable nouns, an executable policy, preconditions, postconditions, and an evaluator. This gives the planner enough semantic structure to reason about tasks and enough execution detail to run on physical robots.

Atomic skills and meta-skills

Atomic skills include Transit, Pick, Place, Support, and Handover. Common compositions are packaged as meta-skills such as PickPlace, PickPlacewSupport, and PickHandoverPlace, so a high-level assembly step can expand into executable robot actions.

Planning, execution, and improvement

The planner grounds semantic assembly steps into robot-object-skill assignments. During execution, the system logs trajectories, forces, camera observations, and skill labels. Those logs are then used to diagnose failures and improve future skill execution.

Pipeline

1. Specify an assembly task

The user can provide a structured task directly, or the system can extract an assembly task from a human demonstration video. The video pipeline performs preprocessing, token reduction, multimodal prompting, and schema-constrained generation.

2. Plan over the Skill Graph

The planner searches for feasible meta-skills, robot assignments, and object choices. It uses preconditions such as stability, reachability, collision constraints, and available inventory.

3. Execute with the bimanual robot system

The grounded sequence is executed by two Yaskawa GP4 robots. APEX-MR converts the planned sequence into an asynchronous execution graph so safe parts of the task can run in parallel.

Per-skill trajectory visualizations

4. Collect data and improve skills

Each execution produces structured logs. These logs support pick/place postcondition checks, structural anomaly detection, and future failure-aware planning.

Demos

Demo 1: Human demonstration to robot execution

A human assembly video is converted into a robot-executable task. The robot execution follows the extracted task sequence while grounding each step through the Skill Graph.

Demo 2: Long-horizon LEGO assembly tasks

The system is evaluated on four LEGO structures with different lengths and geometry: Faucet, Fish, Vessel, and Guitar.

Faucet assembly

Full robot execution of the 14-brick Faucet structure.

Demo 3: Failure recovery

When a pick or place step fails mid-assembly, the system detects the failure through its evaluators and re-attempts the skill automatically without restarting the full task.

Demo 4: Skill improvement after deployment

After adding skill improvements, the system succeeds on all evaluated designs in the reported trials and increases survival length for longer structures.
